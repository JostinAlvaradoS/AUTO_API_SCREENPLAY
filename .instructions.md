# Instrucciones de IA: Especialista en QA (API Screenplay Rest)

Eres un experto en pruebas de API REST utilizando **Screenplay** y **Rest-Assured**.

## Configuración de Entorno (IMPORTANTE)
Las URLs de los servicios están configuradas en `src/test/resources/serenity.properties`.
- **Inventory:** http://localhost:50002
- **Catalog:** http://localhost:50001
- **Ordering:** http://localhost:5003

## Contexto de Negocio
- **Fuente de Verdad:** `shared-specs/specs/001-ticketing-mvp/spec.md`
- **Contratos:** `shared-specs/contracts/openapi/inventory.yaml`

## Reglas de Implementación Técnica
1. **Actor de API:** El actor (ej: `Sistema`) debe tener la habilidad `CallAnApi.at(EnvironmentSpecificConfiguration.from(Serenity.getEnvironmentVariables()).getProperty("restapi.inventory.base.url"))`.
2. **Estructura de Carpetas:**
   - Features: `src/test/resources/features/`
   - Step Definitions: `co.com.ticketing.api.stepdefinitions`
   - Runners: `co.com.ticketing.api.runners`
3. **CRUD Completo:** Implementa un único Scenario que encadene:
   - **POST:** /api/v1/reservations (Crear recurso).
   - **GET:** /api/v1/reservations/{id} (Validar).
   - **DELETE:** /api/v1/reservations/{id} (Limpiar).
4. **Validación de Esquema:** Asegura que las respuestas coincidan con los tipos de datos en el YAML compartido.
5. **Semántica:** Usa `SeeThat` y `ResponseConsequence` para las validaciones de códigos de estado (201, 200).

## Requerimiento de Taller
- Validar el flujo CRUD de la API de Reservas según los contratos de la carpeta compartida.
